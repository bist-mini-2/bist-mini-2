RAG 청크 크기 및 유사도 검색 아키텍처 분석

작성일: 2026. 06. 10 담당 도메인: Computer Science

1. 연구 배경 및 필요성

대규모 학술 및 전문 문헌 대상 RAG(Retrieval-Augmented Generation) 시스템 설계에서 적절한 청크 분할(Chunking) 모델의 선정과 고속 유사도 검색 성능 간의 상관관계는 전체 시스템의 품질을 결정하는 가장 핵심적인 파라미터이다. 정보 유실을 방지하고 질의 맥락의 정확도를 개선하기 위해 3대 학술 도메인(의학, 컴퓨터과학, 자연과학) DB를 기반으로 실시간 인라인 인용이 가능한 지식 에이전트 구축 방안에 대해 분석한다.

2. 핵심 요약 및 연구 성과

  • 500자 단위 청크 최적화: 텍스트 분할 시 500자(50자 중첩) 스키마를 적용하는 것이 단어의 의미적 결합 및 맥락 유실 방지에 최적의 밸런스를 보인다.
  • 유사도 매트릭스 활용: pgvector DB에서 코사인 유사도 점수와 유클리드 거리를 적용하여 청크 간의 유사도 행렬을 고속 추출한다.
  • 실시간 스크래핑 보강: 로컬 지식의 시간적 한계를 극복하기 위해, Tavily API 및 BeautifulSoup을 활용하여 ArXiv/PubMed 등의 최신 외부 논문 abstracts를 실시간 크롤링하여 컨텍스트에 보완한다.

3. 연구 한계점 및 제언

본 리포트 분석에서 사용한 SCIDOCS 데이터셋은 정적인 학술 초록 위주이므로, 테이블(Table) 정보나 복잡한 수식이 포함된 완전 PDF 문서(Full Text) 파싱 시 일부 정밀도가 하락할 여지가 존재한다. 이를 해결하기 위해 향후 Pydantic Structured Output 모델 설계를 고도화하여 수식 및 표 구조 데이터를 정형 JSON 스키마로 분리 래핑하여 적재 및 검색하는 2단계 파이프라인 개발을 적극 권장한다.
