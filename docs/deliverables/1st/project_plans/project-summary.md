# [프로젝트 정의서] 논문 AI 에이전트 채팅 플랫폼 (Paper Agent Chat Platform)

본 문서는 팀 프로젝트의 최종 선정 주제인 **'논문 AI 에이전트 채팅 플랫폼'**에 대한 개요 및 주제 선정 배경을 정리한 정의서입니다.

---

## 📌 1. 프로젝트 개요

- **프로젝트명**: 논문 AI 에이전트 채팅 플랫폼 (Paper Agent Chat Platform)
- **핵심 목표**: 사용자가 입력한 연구 주제 또는 질문에 대해 관련 논문을 에이전트 기반으로 검색, 분석, 채팅하고 인용 관계를 고려한 종합 문헌 조사 리포트를 자동으로 생성하는 시스템 구축
- **기술 스택**:
  - **프론트엔드**: Next.js
  - **백엔드**: FastAPI (LangChain / LangGraph 포함)
  - **데이터베이스**: PostgreSQL (pgvector 지원)

---

## 🎯 2. 주제 선정 배경 및 기준 충족 여부

팀의 주제 선정 기준을 다음과 같이 완벽히 충족하여 최종 주제로 결정되었습니다.

1. **RAG 결과물의 LLM 활용**:
   - 검색된 논문 초록(Abstract) 및 본문 청크들을 RAG로 가져와 LLM이 논리적 요약 및 인용 관계 분석을 수행하는 데 적극적으로 활용합니다.
2. **기구축 데이터셋 활용 (자체 제작 최소화)**:
   - 기구축된 **Kaggle ArXiv 학술 논문 메타데이터셋**을 RAG 및 검색 엔진의 원천 소스로 활용합니다.
   - 외부 API 호출 제한 및 비용 부담 없이 데이터셋 전체를 로컬/데이터베이스에 벌크 적재하여 사용하며, 특히 성능 테스트 및 개발 가속화를 위해 3대 도메인별 2만 건 이하의 경량화된 세부 카테고리 필터링 데이터를 채택하여 RAG 최적화를 수행합니다.
3. **효율적인 RAG Retriever 구성**:
   - 법률 데이터셋과 달리 복잡한 튜닝 없이도, **강의에서 학습한 유클리드 거리 또는 코사인 유사도 기반 pgvector 검색**만으로 의미 있는 수준의 검색 성능을 낼 수 있습니다.
4. **팀원별 Full-Stack 역할 분담**:
   - 기능 단위를 기준으로 나뉘어, 각 팀원이 담당 기능의 프론트엔드(Next.js)와 백엔드(FastAPI) 구현을 모두 책임지는 방식으로 진행합니다.
