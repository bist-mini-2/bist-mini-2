# LangChain & FastAPI 백엔드 연동 구현 체크리스트

이 문서는 `docs/langchain-v5.pptx.pdf`에서 다룬 핵심 기능들을 FastAPI 백엔드에 연동하기 위한 팀원용 체크리스트입니다. 구체적인 코드 구현체보다는 기능 단위의 요건과 엔드포인트 사양을 중심으로 정리했습니다.

---

## 🛠️ 1. 개발 환경 및 공통 모듈 구성
백엔드 프로젝트가 공통으로 가질 예외 처리, 로깅, DB 연결 등 기초 설정을 다룹니다.

- **프로젝트 기초 설정**
  - [ ] 가상환경(Python 3.12) 및 개발 도구(`.settings.json`, `.env` 등) 세팅
  - [ ] `main.py` 진입점 및 의존성 라이브러리(`requirements.txt`) 관리
- **공통 모듈 개발**
  - [ ] **전역 예외 처리기**: HTTP 예외, 유효성 검사 예외 및 일반 예외에 대한 일괄 응답 포맷 구축
  - [ ] **비동기 DB 연결 풀**: PostgreSQL 및 SQLAlchemy 비동기 세션 생성을 위한 의존성 주입(DI) 구조 수립
  - [ ] **커스텀 로깅 콜백**: LLM 요청 프롬프트와 응답 메시지를 가독성 있게 출력하는 로깅 핸들러 추가
- **FastAPI 앱 설정 및 수명 주기 관리**
  - [ ] 앱 시작/종료 시 DB 연결 풀을 안전하게 열고 닫는 라이프사이클(`lifespan`) 핸들러 구현
  - [ ] 정적 파일(/static) 마운트 및 HTML 템플릿 연동
  - [ ] Uvicorn 실행 시 중복 로그 방지 및 레벨별 컬러 로깅 설정
  - [ ] CORS(Cross-Origin Resource Sharing) 보안 정책 미들웨어 추가

## 💬 2. 텍스트 채팅 (Text Chat)
기본적인 LLM 텍스트 추론 및 결과 반환 기능을 구현합니다.

- [ ] **기본 대화 API (`POST /chat-model`)**: 사용자의 단발성 질문에 대해 모델(`gpt-4o-mini`)이 완성형 답변을 한 번에 반환
- [ ] **스트리밍 대화 API (`POST /chat-model-stream`)**: 실시간 타자 효과를 주기 위해 JSON 스트리밍(`application/x-ndjson`) 방식으로 청크 데이터를 전송

---

## 🧠 3. 메시지 처리 및 프롬프트 기법 (Advanced Prompts)
다양한 프롬프트 엔지니어링 기법을 백엔드 비즈니스 로직에 결합합니다.

- [ ] **시스템 페르소나 설정 (`POST /chat-with-system`)**: 시스템 역할을 부여하여 맞춤형 어조(예: 친절한 개발자)로 답변하도록 프롬프트 구성
- [ ] **Few-Shot 프롬프트**: 프롬프트 안에 입력-출력 예시를 포함하여 정형 포맷(JSON 등)에 맞춰 정확하게 답변하도록 가이드
- [ ] **스탭-백(Step-Back) 프롬프트**: 복잡한 질문 수신 시 이를 여러 하위 질문으로 분해한 뒤, 순차적으로 정보를 결합하여 최종 응답을 유도
- [ ] **생각의 사슬 (Chain of Thought, CoT)**: "단계별로 생각하기" 기법과 설명 예시를 제공하여 수학/논리적 문제 해결 성능 강화
- [ ] **자기 일관성 (Self-Consistency)**: 동일한 판별형 질문을 여러 차례 동시에 호출한 뒤, 다수결을 통해 가장 높은 빈도의 결과를 반환

---

## 📊 4. 구조화된 출력 (Structured Output)
자연어로부터 원하는 정보만 추출하여 고정된 스키마 객체(JSON)로 자동 반환합니다.

- [ ] **추출 대상 Pydantic 스키마 정의**
  - [ ] 개인정보 구조화 (`name`, `age`, `email`)
  - [ ] 영화 정보 구조화 (`title`, `year`, `director`, `plot`)
  - [ ] 피자 주문 정보 구조화 (`size`, `type`, `ingredients`)
- [ ] **구조화 응답 API 구현**
  - [ ] **영화 정보 추출 (`POST /structured-output-movie`)**: 자연어 영화 소개에서 정보만 뽑아 영화 객체 형식으로 응답
  - [ ] **피자 주문 분석 (`POST /structured-output-pizza`)**: 비정형 주문 텍스트를 파싱하여 정확한 주문서 객체로 반환

---

## 🤖 5. 에이전트 구축 (Agent)
도구 호출과 판단을 자율적으로 수행하는 AI 에이전트를 모듈화하여 백엔드 서비스로 래핑합니다.

- [ ] **에이전트 서비스 모듈화**: LangChain의 `create_agent`를 활용하여 독자적인 판단 루프를 도는 에이전트 클래스 작성
- [ ] **에이전트 실행 API**
  - [ ] 단발성 비동기 에이전트 대화 엔드포인트 구현
  - [ ] 스트리밍 형식의 에이전트 대화 엔드포인트 구현 (토큰 단위 실시간 전송)
- [ ] **구조화된 출력 에이전트 API (`POST /agent-structured-output`)**: 도구 호출을 거쳐 최종 결과를 지정된 Pydantic 구조로 반환

---

## 💾 6. 대화 히스토리 및 컨텍스트 관리 (Chat History)
에이전트가 사용자와의 대화 맥락을 기억하도록 히스토리 백엔드를 연동합니다.

- [ ] **메모리 기반 대화 세션 관리**: 테스트 및 개발용 RAM 저장소 기반 대화 메모리 연동
- [ ] **자동 대화 요약 미들웨어**: 설정된 토큰 한도를 초과할 경우 오래된 대화는 핵심 내용만 요약 압축하고 최신 대화만 그대로 유지
- [ ] **대화 조회 및 삭제 API**
  - [ ] 대화 식별자(Thread ID) 기준의 대화 기록 포맷 변환 및 조회 API
  - [ ] 특정 스레드의 대화 히스토리를 데이터베이스에서 완전 삭제하는 API
- [ ] **데이터베이스 기반 영구 저장소**: 실제 프로덕션 적용을 위해 `PostgresSaver`를 연동하고 스키마 자동 구축 및 비동기 인스턴스 초기화 지원

---

## 🛠️ 7. 도구 연동 (Tool Calling)
에이전트가 외부 데이터 검색 및 하드웨어 제어 등을 수행하도록 다양한 도구(Tool)를 함수화하여 제공합니다.

- [ ] **기본 알람/시간 도구**: 현재 서울 시간 조회 및 24시간 형식 등의 오류 보정이 들어간 알람 설정 기능
- [ ] **멀티모달 이미지 분석 및 제어**: 차량 이미지를 입력받아 번호판을 파싱하고, 등록 여부 검증 후 차단막 제어 도구(Up/Down)를 단계별로 호출
- [ ] **로컬 파일 시스템 관리 도구**: 루트 디렉토리 접근 차단 보안 규칙이 적용된 폴더 조회, 파일 생성, 읽기, 쓰기, 복사, 이동, 삭제 도구 세트
- [ ] **인터넷 검색 및 웹페이지 추출**: Tavily API 검색과 함께, 타깃 웹페이지의 불필요한 태그를 지우고 순수 텍스트 본문만 정제해 추출하는 BeautifulSoup 파서 도구 결합
- [ ] **결과 직접 반환 도구 (`return_direct`)**: LLM의 추가 요약/답변 과정 없이 도구의 실행 결과(예: 추천 영화 목록 JSON)를 즉시 클라이언트에 응답
- [ ] **보안 정보 전달을 위한 도구 컨텍스트 (`context_schema`)**: API 키 등 민감한 인증 정보를 LLM에 노출하지 않고 백엔드 내부 도구로 직접 전달해 동작
- [ ] **에이전트 공유 상태 관리 도구 (`state_schema`)**: 에이전트의 작동 흐름 중에 여러 도구가 공유 메모리(State)에 특정 값(사용자 역할, 권한 목록 등)을 비동기로 갱신 및 참조할 수 있는 체계 수립

## 📚 8. 문서 기반 검색 시스템 (RAG)
외부 문서(PDF 등)를 분석하여 해당 정보를 기반으로 정확하게 답변하는 파이프라인을 구축합니다.

- [ ] **문서 로드 및 청크 분할**: PDF 파싱, 메타데이터 부여 및 텍스트 500자 단위 분할
- [ ] **SQLAlchemy 연동 비동기 벡터 DB 적재**: `text-embedding-3-large` 모델을 이용해 쪼개진 텍스트 벡터를 PostgreSQL(pgvector)에 비동기로 저장
- [ ] **PDF 업로드 및 임베딩 API (`POST /pdf-embedding`)**: PDF 파일을 업로드받아 유효성을 검증하고, 백그라운드에서 임베딩 적재 작업을 한 번에 실행
- [ ] **유사도 기반 문서 검색 API (`POST /similarity-search`)**: 질의어와 코사인 유사도가 가장 높은 문서 청크와 가중치 점수를 포맷팅하여 검색
- [ ] **RAG 에이전트 API (`POST /agent-rag`)**: 헌법 지식 데이터베이스 조회 도구를 내장하고, 출처(조항, 페이지 등)를 표기하며 도메인 지식 밖의 질문은 대답을 회피하도록 제어된 에이전트 구축

---

## 🤝 9. 멀티 에이전트 시스템 (Multi-Agent System)
복잡한 작업을 서로 다른 전문 영역의 에이전트들이 분담하여 협업하고 조율하는 워크플로우를 구현합니다.

- [ ] **공유 상태(Shared State) 설계**: 모든 노드(에이전트)들이 대화 내역, 분석 결과, 조회 데이터를 취합해 갈 수 있는 데이터 키-밸어 구조 설계
- [ ] **고객 지원 오케스트레이션 그래프 구축**
  - [ ] 문의를 분석하여 핵심 정보를 분류하는 분석 노드 구성
  - [ ] 회원 정보와 지식 베이스를 병렬로 검색하여 하나로 뭉치는 취합(Gather) 노드 구성
  - [ ] 문의 유형(기술지원, 주문/배송, 환불/교환, 계정관리, 일반문의)에 따라 최적의 하위 전담 에이전트로 상태를 전달해 처리하는 조건부 라우팅(Conditional Edges) 설정
- [ ] **그래프 실행 및 구조 시각화 API**
  - [ ] 멀티 에이전트 워크플로우를 기동하여 최종 완성 응답을 받아 반환하는 API (`POST /customer-support`)
  - [ ] 구성된 협업 그래프의 구조를 시각화한 PNG 이미지 스트림을 반환하는 디버깅용 API (`GET /graph-structure`)
